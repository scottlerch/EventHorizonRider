# App Store screenshots

Generated by the **iOS App Store Screenshots** GitHub Actions workflow
([`.github/workflows/ios-screenshots.yml`](../.github/workflows/ios-screenshots.yml)): it
builds the Simulator app, boots a 6.9" iPhone and a 13" iPad, launches the game, and captures
the screen. The game is landscape-only, so the Simulator's portrait captures are rotated 90°
counter-clockwise to true landscape afterwards. The iPhone capture is then downscaled from the
6.9" native size (2868 × 1320) to **6.7" (2778 × 1284)** — the size App Store Connect's iPhone
screenshot slot accepts (it rejects 6.9" dimensions).

| File | Device | Pixels | App Store slot |
| --- | --- | --- | --- |
| `iphone-6.7-inch-*.png` | iPhone 6.7" | 2778 × 1284 | 6.7" iPhone (covers all iPhone sizes) |
| `ipad-13-inch-*.png` | iPad Pro 13" | 2752 × 2064 | 13" iPad (also covers 12.9") |

These two device sizes satisfy the current required screenshot sets for an iPhone + iPad app.
The `-1` / `-2` files are alternate frames of the title screen; one of each is enough to submit.

Upload them under **App Store Connect → App Store → 1.3 → Previews and Screenshots**, and
remove the obsolete 2016 captures (3.5", 4", and 9.7" iPad display types).

To regenerate after art/UI changes: re-run the workflow, rotate the raw PNGs 90° CCW, and
downscale the iPhone ones to 2778 × 1284.
